# Codex引き継ぎ用MD
## プロジェクト名（仮）
**アスクレピオス学園 - 星とことばの診療録 -**

---

## 1. このドキュメントの目的
このMDは、Codexにそのまま渡してスマホ向けゲームのMVPを実装しやすくするための**企画仕様 + 実装方針 + タスク分解**をまとめた引き継ぎ文書である。

対象は**α世代に近い感性を持つ女子高校生**。ただし教材の出発点は**医療系専門学校の入学前教育**なので、
- 軽く遊べる
- 続きが気になる
- 推し活できる
- でも中身は読解・思考・要約に触れられる

というバランスを最優先にする。

---

## 2. 元教材から読み取るべき設計方針
元教材は大きく2本ある。

### A. 読解教材の特徴
- 題材は**「医神アスクレピオス」**の物語
- 学習目的は**医療を目指す者としての基本姿勢**と**日本語文章理解の向上**
- 構成は、通読 → イメージ理解 → 文脈理解 → 漢字力と意味理解 → 要約力 → 深層理解 → 推測力/論理力 → 精読 → 再要約
- 「メモを取ってよい」「本文を再参照してよい」「各Stepの最後にQuestionがある」という、**反復理解型**の構成

### B. 思考力教材の特徴
- 1問完結型の論理問題が30問
- 内容は、並べ替え、真偽判定、座席、階段、靴下、暗号、リーグ戦、勤務表、曜日、集合、図形、場合の数など
- **短時間で解けるミニゲーム化**に向いている

### C. ゲーム化で採用すべき解釈
したがって、このゲームは次の二層構造にする。

1. **メイン層**: 神話ベースのストーリーADV（読解教材を変換）
2. **サブ層**: 日替わりの短い論理ミニゲーム（思考力教材を変換）

この二層構造を守ること。

---

## 3. プロダクトビジョン
### 一言でいうと
**「読むほど世界にハマり、解くほど推しが育つ、神話×医療×ことばの放課後ゲーム」**

### プレイヤー体験の核
- 毎日3〜5分で遊べる
- 物語の続きが気になる
- 問題を解くと推しとの会話や演出が増える
- 勉強している感を薄め、ゲーム体験を前面に出す
- ただし世界観の根には「生に奉仕する医療」という主題を残す

## 4. ターゲット
### メインターゲット
- 女子高校生
- 神秘的な世界観、推しキャラ、短時間プレイ、日課化しやすい設計を好む層

### サブターゲット
- 医療系進学を考えている学生
- 読解や思考問題をゲームなら触れやすい層

### 刺さる要素
- 星座、神殿、薬草、白/黒カラス、蛇の杖
- 美形キャラ、少し切ない神話、秘密、ルーム、ボイス、日記
- 1日1回の気軽な達成感

## 5. ゲームコンセプト
プレイヤーは「アスクレピオス学園」の新入生。
放課後に**診療録**と呼ばれる神話記録を読み、仲間たちと一緒に過去の事件・診療・選択をたどる。

プレイヤーは以下を進める。
- 物語を読む
- その回に対応する読解系ミッションを解く
- 1日1〜3問の思考ミニゲームを解く
- 報酬で推しキャラ、星座ルーム、薬草園を育てる

---

## 6. 世界観ルール
### 基本トーン
- 幻想的
- 透明感
- 神秘的
- 少し切ない
- 上品

### 避けること
- 医療行為の過度なリアル表現
- グロ描写
- 重すぎる死の演出
- 過激な対人競争
- 「課金しないと学習不能」な設計

### 重要ルール
このゲームは**神話・教育モチーフのフィクション**であり、現実の医療判断や治療法の案内を目的としない。
古代の薬草や処置は世界観素材として扱い、**現代医療の助言として見えないUI/文言**にすること。

---

## 7. コアループ
1. ログイン
2. 今日のメインストーリーを1話読む（1〜2分）
3. ストーリーに紐づく読解ミッションを1〜2問解く（1〜2分）
4. 日替わり思考ミニゲームを1問解く（30〜60秒）
5. 報酬で推し/ルーム/薬草園を強化
6. 次回の予告・未読通知・キャラメッセージを見て離脱

### 1セッションの想定
- 合計3〜5分
- 長くても8分以内

---

## 8. 読解教材のゲーム変換ルール
### 8-1. イメージ理解
紙の問題 → ゲームでは以下に変換
- シーンに合う一枚絵を選ぶ
- 表情を選ぶ
- 持ち物を選ぶ
- 誰の視点か当てる

### 8-2. 文脈理解
紙の問題 → ゲームでは以下に変換
- 吹き出し差し込み
- 会話順並べ替え
- 欠けた一文を選ぶ

### 8-3. 漢字力・意味理解
紙の問題 → ゲームでは以下に変換
- ことばのかけらを集めて語を完成
- 語義マッチング
- ひらがな→漢字の選択式
- 用語ミニ辞典解放

### 8-4. 要約力
紙の40字/400字課題 → ゲームでは以下に変換
初期MVPでは自由記述を最小限にし、以下の順で段階化する。
1. キーワード選択
2. 文パーツ並べ替え
3. 20〜40字の短文入力

### 8-5. 深層理解・推測
紙の問題 → ゲームでは以下に変換
- キャラの本音を選ぶ
- なぜそうしたかを推理する
- 医療観・人生観に触れる選択肢を選ぶ

### 8-6. 論理力
読解教材内の論理要素は、ストーリー分岐ではなく**推理演出付きの単発問題**として出す。

---

## 9. 思考力教材のゲーム変換ルール
思考力教材は**日替わりミニゲーム群**として実装する。

### 対応表
- 並べ替え問題 → ストーリー復元パズル
- 真偽判定 → うそつき探し
- 円卓/座席 → 配置パズル
- 階段/時間 → クイック計算チャレンジ
- 靴下問題 → 最小手数ひらめき
- 暗号 → 神話コード解読
- リーグ戦 → 勝敗整理ボード
- 勤務表/当番 → スケジュール推理
- 曜日 → カレンダーパズル
- 集合 → 条件カード整理
- 図形 → 神殿ギミック解除
- 場合の数 → 調合レシピ数え上げ

### ミニゲームの設計原則
- 1問30〜90秒で終わる
- 失敗しても即再挑戦できる
- 1日で全問解かせない
- デイリーでランダム出題する

---

## 10. リテンション設計
### 10-1. ログイン理由は「石」より「続き」
主報酬は以下。
- 次話先読みキー
- キャラ日記
- 限定会話
- ルーム家具
- 薬草園の種
- スチル解放素材

### 10-2. 連続ログイン切れで萎えない
- 週3〜4回でも十分進む
- 休んだぶんを回収できる「追いつきチケット」あり
- 連続ログイン報酬が切れても価値が極端に下がらない

### 10-3. PvPは入れない
競争より協力を優先。
- 全体イベント
- フレンドへハート送付
- みんなで神殿修復

### 10-4. 推し導線を強化
- 好感度で個別会話解放
- ルーム訪問
- 制服/私服スキン
- 期間限定メッセージ

## 13. 学習設計ポリシー
### 重要
「学習ゲーム」感を前面に出さない。
見た目はゲーム、内部で読解・要約・思考を処理する。

### 難易度設計
- 初回は必ず解ける易しめ
- 同じ概念を形式を変えて反復
- 間違えたときは即答えではなくヒントを段階表示

### 要約の扱い
自由記述は離脱要因になりやすいので、MVPでは以下に限定。
- 20字要約
- 3つのキーワード選択
- 文パーツを並べて要点を作る

## 16. 推奨技術スタック
Codexで最短で形にするなら以下を推奨。

### フロント
- **Expo + React Native + TypeScript**

### 状態管理
- Zustand

### 画面遷移
- Expo Router もしくは React Navigation

### データ保存
- AsyncStorage

### UI
- React Native Paper または Tamagui なしのシンプル自前UI

### アニメーション
- Reanimated はMVPなら不要
- 軽いフェードだけでもよい

### データ形式
- JSONベース
- story chapters / questions / characters / rewards を静的JSONで管理

### 理由
- Codexが生成しやすい
- ローカル完結で動かせる
- スマホ実機確認が容易

## 20. UXルール
- テキストは縦長スマホで読みやすく、1吹き出し短め
- 1画面に情報を詰め込みすぎない
- 正答時は必ず小さな気持ちよさを返す
- 不正解時は責めない。ヒントを出して再挑戦へ誘導
- 説教感を出さない

### 演出キーワード
- ガラス
- 月光
- 星屑
- 白金
- 薬草の緑
- 柔らかい発光

## 25. 補足: 企画として絶対に守る芯
このゲームは、ただの神話ゲームでも、ただの学習アプリでもない。

**「物語を読み、考え、選び、少しだけ人の生き方に触れる」**
という体験が芯である。

読解教材で重要なのは、単なる知識暗記ではなく、
- 何を読み取るか
- なぜそう判断するか
- 医療は何のためにあるか

に触れている点である。

そのため、MVPでも少なくとも1回は
- 延命
- 患者本人の意思
- 生に奉仕する医術

に触れる会話イベントを入れること。

ただし演出は重くしすぎず、プレイヤーに「考えたくなる余白」を残す。
