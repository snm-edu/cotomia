# CLAUDE.md — 医神アスクレピオス 読解力クエスト

## プロジェクト概要

看護専門学校（札幌看護医療学院）の入学前教育用スマホゲーム。
ギリシャ神話「医神アスクレピオス」の読解力トレーニング（紙ベースPDF・全19問・5Step構成）を、
**すごろく×クイズ×カード収集のハイブリッド型ゲーム**に変換するプロジェクト。

### ターゲット
- 看護学生（18〜22歳、女性が多い）
- 説明書不要で直感的に遊べることが最重要
- スマホ縦画面を前提としたUI

### 世界観・デザイン
- パステル×くすみカラー（清潔感ある医療イメージとの親和性）
- ギリシャ神話モチーフ（神殿、月桂冠、蛇、星座）
- かわいい系とおしゃれ系のハイブリッド

## ゲームメカニクス実装メモ

### 1. スワイプマッチング（問1, 6, 13）
- シーン描写（将来的にイラスト）を見て、対応する物語の節を選択
- 正解で緑のscale(1.03)アニメーション
- 全問正解でコンボボーナス（+50 XP）

### 2. パズルピース穴埋め（問2, 7, 14, 17）
- 文章中の空欄に接続詞をはめ込む
- ピル型ボタンのタップ選択
- 正解でscale(1.06) + 色変化
- ヒントテキスト付き

### 3. 漢字バトル（問3, 10）
- タイマー付きタイムアタック（45〜60秒）
- ひらがな表示→4択から正しい漢字を選択
- 選択肢はシャッフルされる
- タイマーバーが色変化（緑→黄→赤）

### 4. 要約チャレンジ（問4, 11, 16）
- テキストエリアに文字数制限付きで入力
- リアルタイム文字数カウンター
- 制限超過で赤色警告
- 将来的にClaude APIでAI採点

### 5. 並べ替えタイムライン（問14）
- 出来事カードを↑↓ボタンで並べ替え
- 確定ボタンで正誤判定
- 正解位置は緑、不正解は赤の数字バッジ

### 6. ボス戦（Step 5）
- ゼウスのHPバー（100% → 0%）
- 5問連続クイズ、正解でHP -20%
- HPバーにグラデーション＋アニメーション
- 暗い背景（神殿の夜空風）で演出差別化

### フォント
- 優先: `'Noto Sans JP', 'Hiragino Kaku Gothic ProN', sans-serif`
- 将来的にGoogleフォント（Zen Maru Gothic等）への変更を検討

### コンポーネント共通ルール
- border-radius: 12〜16px（カード）, 20〜24px（ボタン）, 8〜10px（小要素）
- box-shadow: `0 2px 12px rgba(0,0,0,0.04)`（カード影）
- transition: `all 0.2s`〜`0.3s`（インタラクション）
- 正解演出: scale(1.03〜1.06) + 背景色変化 + 1000〜1200msで自動進行

## 既存プロジェクトとの関係

### 「医神アスクレピオス」RPG版（別プロジェクト）
- コンボバトル型ビジュアルノベルRPG
- ダークな神話美学、Final Fantasy風バトル演出
- 2年次以降の上級読解力＋医療知識統合向け
- **本プロジェクトとの棲み分け**: ガチャ・バトル・カード収集のメカニクスはRPG側に予約済み
  → 本ゲームではカード「収集」はOKだが「バトルで使う」メカニクスは入れない

### 「オルガモン図鑑クエスト」（別プロジェクト）
- 解剖学コレクションゲーム（ガチャメカニクス）
- **本プロジェクトとの棲み分け**: ガチャ演出はオルガモン側に予約済み
  → 本ゲームのカード獲得は「問題正解報酬」であって「ランダムガチャ」ではない

## 開発優先度（推奨順序）

### Phase 1: プロジェクト基盤（1-2日）
1. `npm create vite@latest asclepius-quest -- --template react-ts`
2. App.jsxをコンポーネント分割（上記ディレクトリ構成に沿って）
3. TypeScript化
4. Tailwind CSS導入（インラインスタイルからの移行）
5. ローカルストレージで進捗永続化

### Phase 2: 問題データ完全化（1日）
1. PDFから問3/問10の漢字データを完全抽出
2. 問10（文章挿入）の専用UIを実装
3. 問15（文章挿入・Step3版）の実装
4. 接続詞問題の正解を最終確認

### Phase 3: UX強化（2-3日）
1. PWA対応（オフライン対応、ホーム画面追加）
2. 効果音追加（Howler.js or Tone.js）
3. Lottieアニメーション（カード獲得、ボス撃破）
4. ページ遷移アニメーション（framer-motion）
5. ストリーク通知のスケジュール

### Phase 4: ソーシャル機能（2-3日）
1. Firebase Auth（匿名認証 or LINEログイン）
2. Firestore（進捗・スコア保存）
3. ランキング機能（学校・クラス単位）
4. SNSシェア（Wordle風グリッド生成＋共有API）

### Phase 5: AI採点（1日）
1. Claude API連携（要約問題のみ）
2. 採点基準: 要点の網羅度を3段階（★☆☆〜★★★）で判定
3. コスト管理: 1回の採点 ≈ 数百トークン、Haiku使用で低コスト化
